Lab 2.1 - ESO

Start Guide: rotate secret không restart pod

Mục tiêu theo slide mentor: chuyển DB password sang AWS Secrets Manager + External Secrets Operator, K8s Secret cập nhật trong khoảng <= 60s, pod đọc secret qua volume nên không restart.

Code đã có

eso/, argocd/apps/eso.yaml, argocd/apps/eso-config.yaml.

Không commit secret

AWS access key tạo bằng kubectl create secret, không đưa vào Git.

1. Đọc nhanh slide Lab 2.1

  • SecretStore: cấu hình provider AWS Secrets Manager, region ap-southeast-1.
  • ExternalSecret: map secret nguồn /demo/db/password thành K8s Secret db-secret, key password.
  • refreshInterval: đặt 1m để đạt yêu cầu cập nhật trong <= 60s.
  • No restart: pod phải mount secret bằng volume, không dùng env.
  • Thứ tự GitOps: ESO operator/CRD chạy trước, config SecretStore/ExternalSecret chạy sau bằng sync wave.

2. Chuẩn bị cluster

Chạy trong PowerShell. Nếu Docker Desktop chưa mở, mở trước rồi tạo minikube profile.

minikube start -p w10 --driver=docker
kubectl config use-context w10
kubectl get nodes

3. Cài ArgoCD nếu cluster mới

kubectl create ns argocd
kubectl apply --server-side -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
kubectl -n argocd rollout status deploy/argocd-server

Lấy password UI nếu cần chụp ArgoCD evidence:

$pwd64 = kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}"
[Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($pwd64))

4. Luồng A - AWS thật, đúng chuẩn mentor

Quan trọng:

Trước khi apply eso-config.yaml, phải có AWS secret nguồn và Kubernetes secret aws-creds. Nếu thiếu, ExternalSecret sẽ chưa Ready.

4.1. Tạo AWS secret nguồn

Dùng AWS Console hoặc AWS CLI. Tên secret nguồn trong manifest là /demo/db/password.

aws secretsmanager create-secret `
  --name /demo/db/password `
  --secret-string "password-v1" `
  --region ap-southeast-1

Nếu secret đã tồn tại, rotate bằng:

aws secretsmanager put-secret-value `
  --secret-id /demo/db/password `
  --secret-string "password-v2" `
  --region ap-southeast-1

4.2. Apply ESO operator trước

kubectl apply -f argocd/apps/eso.yaml
kubectl get applications -n argocd
kubectl -n external-secrets get pods
kubectl -n external-secrets rollout status deploy/external-secrets

4.3. Tạo namespace và AWS credentials thủ công

kubectl apply -f eso/namespace.yaml
kubectl create secret generic aws-creds -n demo `
  --from-literal=access-key=REPLACE_WITH_AWS_ACCESS_KEY_ID `
  --from-literal=secret-key=REPLACE_WITH_AWS_SECRET_ACCESS_KEY

4.4. Apply ESO config qua ArgoCD

kubectl apply -f argocd/apps/eso-config.yaml
kubectl get applications -n argocd
kubectl get secretstore -n demo
kubectl get externalsecret -n demo

4.5. Kiểm tra secret đã sync

kubectl describe secretstore aws-secretsmanager -n demo
kubectl describe externalsecret db-secret -n demo
kubectl get secret db-secret -n demo

Decode secret trên Windows PowerShell:

$b64 = kubectl get secret db-secret -n demo -o jsonpath="{.data.password}"
[Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($b64))

4.6. Chứng minh pod không restart khi rotate

kubectl get pod secret-reader -n demo -o wide
kubectl get pod secret-reader -n demo -o jsonpath="{.metadata.creationTimestamp}{`n}"
kubectl logs secret-reader -n demo --tail=20

Rotate trên AWS, chờ 60 giây, rồi chạy lại:

aws secretsmanager put-secret-value `
  --secret-id /demo/db/password `
  --secret-string "password-v2" `
  --region ap-southeast-1

Start-Sleep -Seconds 70

$b64 = kubectl get secret db-secret -n demo -o jsonpath="{.data.password}"
[Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($b64))
kubectl logs secret-reader -n demo --tail=20
kubectl get pod secret-reader -n demo -o jsonpath="{.metadata.creationTimestamp}{`n}"
Kỳ vọng nghiệm thu:

Giá trị secret đổi sang password-v2, log pod đọc được value mới, creationTimestamp/AGE của pod không đổi.

5. Luồng B - Fake provider local khi chưa có AWS

Fallback local:

Luồng này dùng secret-store-fake.yaml để chứng minh pattern. Không apply argocd/apps/eso-config.yaml trong luồng fake, vì ArgoCD self-heal sẽ đưa ExternalSecret quay lại AWS store.

5.1. Cài ESO operator bằng ArgoCD

kubectl apply -f argocd/apps/eso.yaml
kubectl -n external-secrets rollout status deploy/external-secrets

5.2. Apply config fake trực tiếp

kubectl apply -f eso/namespace.yaml
kubectl apply -f eso/secret-store-fake.yaml
kubectl apply -f eso/external-secret.yaml
kubectl patch externalsecret db-secret -n demo --type merge -p '{"spec":{"secretStoreRef":{"name":"fake-secrets","kind":"SecretStore"},"data":[{"secretKey":"password","remoteRef":{"key":"/demo/db/password","version":"v1"}}]}}'
kubectl apply -f eso/secret-reader-pod.yaml

5.3. Kiểm tra value v1

kubectl get secretstore fake-secrets -n demo
kubectl get externalsecret db-secret -n demo
$b64 = kubectl get secret db-secret -n demo -o jsonpath="{.data.password}"
[Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($b64))
kubectl logs secret-reader -n demo --tail=10
kubectl get pod secret-reader -n demo -o jsonpath="{.metadata.creationTimestamp}{`n}"

5.4. Rotate fake v1 sang v2

kubectl patch externalsecret db-secret -n demo --type merge -p '{"spec":{"data":[{"secretKey":"password","remoteRef":{"key":"/demo/db/password","version":"v2"}}]}}'
Start-Sleep -Seconds 70
$b64 = kubectl get secret db-secret -n demo -o jsonpath="{.data.password}"
[Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($b64))
kubectl logs secret-reader -n demo --tail=20
kubectl get pod secret-reader -n demo -o jsonpath="{.metadata.creationTimestamp}{`n}"

6. Evidence cần chụp cho mentor

  • kubectl get pods -n external-secrets: ESO operator Running.
  • kubectl describe secretstore aws-secretsmanager -n demo: AWS SecretStore Ready nếu dùng AWS.
  • kubectl describe externalsecret db-secret -n demo: ExternalSecret synced.
  • Decode db-secret/password trước và sau rotate.
  • kubectl get pod secret-reader -n demo: AGE hoặc creationTimestamp không đổi sau rotate.
  • kubectl logs secret-reader -n demo --tail=20: pod đọc value mới từ mounted volume.
  • Secret scan trong repo không có AWS key/private key thật.

7. Lệnh chứng minh repo sạch secret

8. Troubleshooting nhanh

SecretStore/ExternalSecret no matches for kind

ESO CRD chưa có. Chờ external-secrets rollout xong rồi apply config.

ExternalSecret chưa Ready

Kiểm tra aws-creds, region, IAM quyền secretsmanager:GetSecretValue, và tên secret /demo/db/password.

Pod không đọc được file

Kiểm tra Secret db-secret đã tồn tại chưa. Pod mount path là /mnt/db/password.

Fake provider bị quay về AWS

Bạn đã apply eso-config.yaml. Với fake local, test trực tiếp bằng kubectl hoặc tạm không sync app đó.

File này được tạo từ yêu cầu Lab 2.1 trong slide w10_afternoon_secrets_supply_chain.html và khớp với manifest hiện tại trong repo.
